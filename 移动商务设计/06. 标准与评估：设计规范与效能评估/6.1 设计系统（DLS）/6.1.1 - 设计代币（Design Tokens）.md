## 📄 6.1.1 - 设计代币（Design Tokens）.md

标签：#设计系统 #Design Tokens #实操规范 #团队协作

### 💎 四级子项：定义与核心逻辑：设计原子的代码化映射

**设计代币（Design Tokens）** 是设计系统中最底层的**原子级视觉属性**（如颜色、字体大小、间距、圆角等）的**命名实体**。它们将设计决策从设计软件（如 Figma）桥接到所有开发环境（Web, iOS, Android），确保设计和代码中的所有基础样式都保持**单一事实来源（Single Source of Truth, SSOT）**。

**核心目标：**

1.  **实现跨平台一致性：** 确保品牌色在所有平台和主题下（如亮/暗模式）都表现一致。
2.  **提高维护效率：** 只需要修改代币的值，就可以全局更新所有引用该代币的组件。
3.  **促进设计与开发协作：** 消除设计稿与代码之间的“翻译”环节，设计师和开发者使用同一套语言。

### 🏷️ 代币分类与命名规范

代币的命名应遵循**语义化**和**层级化**原则，通常分为**基础代币（Global Tokens）**和**语义代币（Alias/Semantic Tokens）**。

| 代币类型 | 核心用途 | 命名示例（$Type.Group.Subgroup.Variant$） | 示例值 |
| :--- | :--- | :--- | :--- |
| **色彩代币** | 定义具体的色值。 | `$color.brand.primary.default$` | `#FF4400`（淘宝红） |
| | **语义代币：** 用于特定场景。 | `$color.text.button.cta$` | `$color.brand.primary.default$` |
| **排版代币** | 定义字号、行高、字重。 | `$font.size.body.large$` | $16px$ |
| | **语义代币：** 用于特定组件。 | `$font.weight.heading.bold$` | $600$ |
| **尺寸代币** | 定义间距、圆角、边距。 | `$space.spacing.2x$` (8pt 栅格体系中的 $2 \times 8 = 16px$) | $16px$ |
| | **语义代币：** 用于特定场景。 | `$radius.component.card$` | $8px$ |

### 🛠️ 代币在电商设计中的应用

| 代币场景 | 代币名称与语义 | 核心价值 | 关联规范 |
| :--- | :--- | :--- | :--- |
| **主按钮** | `$color.bg.button.cta.default$` | **全局统一**：确保所有页面的“立即购买”按钮都使用完全一致的品牌色。 | [[2.1.1 - 品牌色规范.md]] |
| **深色模式** | `$color.bg.surface.primary.dark$` | **跨模式适配**：亮模式下的白背景自动映射为暗模式下的深灰色。 | [[2.1.4 - 深色模式适配.md]] |
| **卡片圆角** | `$radius.component.card$` | **一致性**：所有商品卡片、会员卡片的圆角（如 $8px$）保持一致，且修改一次全局生效。 | [[3.3.1 - 卡片式设计（Card UI）.md]] |
| **安全间距** | `$space.spacing.screen.horizontal$` | **适配统一**：定义左右两侧 $16pt$ 的安全边距，并应用于所有页面布局。 | [[3.1.2 - 安全距离与适配.md]] |
