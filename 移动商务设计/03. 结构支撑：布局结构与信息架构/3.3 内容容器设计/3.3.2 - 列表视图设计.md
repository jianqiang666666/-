## 📄 3.3.2 - 列表视图设计.md

标签：#布局结构 #容器设计 #信息架构 #效率模式

### 📑 四级子项：定义与核心逻辑：高效、有序的同质化信息排列

**列表视图设计（List View Design）** 是一种将**大量同质化（Homogeneous）信息**以垂直堆叠的行（Row）形式展示的布局模式。列表视图是应用中最常见的界面模式之一，主要用于展示**目录、设置、订单历史**或**结构简单**的商品列表。

**核心目标：**

1.  **高效率浏览：** 通过统一的行高、对齐方式和清晰的分隔线，帮助用户快速扫描和比较信息。
2.  **操作聚焦：** 确保每行（Item）作为一个独立的点击区域，承载清晰的跳转或操作功能。
3.  **信息密度：** 在有限的屏幕空间内，尽可能高效地展示结构化的核心数据。

### 📐 实操规范：列表的结构与元素

列表中的每个条目（Row）都是一个独立的单元，通常由左侧图标、标题、右侧辅助信息和分隔线组成。

| 元素区域 | 核心内容 | 布局原则与规范细节 | 电商应用场景举例 |
| :--- | :--- | :--- | :--- |
| **左侧图标/主图** | 图标、用户头像、商品缩略图 | **图标：** 使用 [[2.3.2 - 图标系统设计.md|线性/面性图标]]，遵循 **[[2.1.1 - 品牌色规范.md|中性色]]** 或 **功能色**。**主图：** 采用统一的方形或 4:3 比例，且必须**左对齐**。 | [[4.4.1 - 个人中心设计.md|个人中心]] 的服务入口、订单列表的商品图。 |
| **标题/描述** | 核心文字信息 | 标题使用 [[2.2.3 - 信息层级排版法.md|主层级字号]]，描述（如商品SKU）使用辅助层级字号。标题和描述必须**左对齐**。 | 订单名称、商品标题。 |
| **右侧辅助信息** | 箭头、数字角标、状态文字 | 常用 **> 箭头**表示跳转，使用 **[[2.1.1 - 品牌色规范.md|中性色]]**。状态文字使用 **[[5.1.2 - 状态变更反馈.md|功能色]]** 突出（如“已完成”）。 | 设置页面的版本号、优惠券数量。 |
| **分隔线（Divider）** | 分隔符 | 使用极浅的 **[[2.1.1 - 品牌色规范.md|中性色]]**（如 #E0E0E0），仅分隔内容主体，不应延伸至左右两侧边界（避免与 **[[3.1.2 - 安全距离与适配.md|安全距离]]** 冲突）。 | 区分设置项、联系人列表。 |

### 🛒 电商应用：列表与卡片的对比

| 特性 | 列表视图（List View） | 卡片式设计（Card UI） |
| :--- | :--- | :--- |
| **信息类型** | **同质化、高密度**（如设置、目录、表格数据） | **异质化、独立性强**（如商品、文章、活动） |
| **视觉边界** | 仅依赖 **分隔线** 或 **极浅阴影** | 依赖 **背景抬升、圆角和阴影** 明显区分 |
| **布局模式** | **单列** 垂直堆叠 | **单列/双列** 网格/瀑布流 |
| **适用场景** | [[4.4.1 - 个人中心设计.md|个人中心]]、[[4.3.2 - 结算页设计.md|结算页]] 价格明细、收货地址列表。 | [[4.2.3 - 商品详情页（PDP）设计.md|PDP]] 推荐模块、[[4.2.2 - 搜索与筛选页设计.md|搜索结果双列]]。 |
