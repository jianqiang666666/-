## 📄 3.3.1 - 卡片式设计（Card UI）.md

标签：#布局结构 #容器设计 #信息架构 #移动端适配

### 📦 四级子项：定义与核心逻辑：独立、可移植的信息容器

**卡片式设计（Card UI）** 是移动端界面中最流行的内容容器模式。卡片（Card）是一个**矩形或圆角矩形**容器，用于封装一组**相关联的信息**（如一张图片、一个标题、价格、一个按钮），使其作为一个独立、可操作的单元存在。

**核心目标：**

1.  **信息独立性：** 卡片通过 **阴影、边框或背景色** 与周围环境和邻近卡片明显区分，遵循 **[[2.2.3 - 信息层级排版法.md|亲密性原则]]**，确保用户快速理解内容分组。
2.  **模块化与可移植性：** 卡片可以灵活地在不同布局（如单列、双列、轮播）和不同页面（首页、列表页、详情页）中复用。
3.  **提升点击效率：** 卡片本身通常是一个 **[[1.1.3 - 拇指法则与热区设计.md|热区]]**，方便用户点击，提升 **CTR**（点击率）。

### 📐 实操规范：卡片的尺寸与视觉表现

| 规范项目 | 核心要求 | 视觉属性与规范细节 | 电商应用场景举例 |
| :--- | :--- | :--- | :--- |
| **基本形态** | **圆角矩形** | **圆角半径（Corner Radius）** 应统一（如 8pt 或 12pt），在 [[2.3.2 - 图标系统设计.md|图标系统]] 中也保持一致。 | 所有商品卡片、文章、优惠券容器。 |
| **视觉抬升** | **阴影/背景色** | **浅色模式：** 使用**柔和的阴影**（Soft Shadow）在 [[3.1.1 - 栅格系统规范.md|网格]] 上微微抬升卡片。**深色模式：** 通过 [[2.1.4 - 深色模式适配.md|亮度差异]] 区分层级。 | 确保卡片与底层画布（Canvas）分离。 |
| **内部布局** | **信息分组** | 遵循 [[2.2.3 - 信息层级排版法.md|信息层级]]，通过 **Padding/Margin** 将卡片内部元素划分区域（如：顶部图片区，底部描述区）。 | 商品图片、标题、价格、CTA 按钮分别占据卡片的不同区域。 |
| **可点击性** | **点击反馈** | 卡片被点击时，应有 **[[5.1.2 - 状态变更反馈.md|清晰的状态变更反馈]]**，如短暂的背景色变暗或轻微的抬升动效。 | 列表页点击商品卡片时的反馈。 |

### 🛒 电商应用：卡片布局与效率

* **商品双列网格（Grid View）：** 在搜索/列表页，卡片是构建 **双列瀑布流** 的基础。双列设计利用了 [[3.1.1 - 栅格系统规范.md|栅格系统]]，实现信息密度和视觉平衡。
* **首页信息流（Feed）：** 首页的信息流由不同内容类型（商品、直播、活动）的卡片构成，这使得 **[[4.2.1 - 首页与金刚区设计.md|信息流]]** 具有高度的灵活性和可插拔性。
* **优惠券与活动卡片：** 优惠券卡片常使用 **[[2.1.2 - 营销色与功能色.md|营销色]]** 背景和特殊的异形切割（如虚线或波浪线）来强化 **[[1.2.3 - 价格锚定视觉化.md|价格锚定]]** 和 **[[1.3.2 - 惊喜感视觉元素.md|惊喜感]]**。
