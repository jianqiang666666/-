## 📄 2.1.2 - 营销色与功能色.md

标签：#色彩系统 #功能色 #实操规范 #交互反馈

### 🚥 四级子项：定义与核心逻辑：具有明确语义的功能性色彩

**营销色与功能色（Marketing & Functional Colors）** 是色彩体系中用于传达**特定操作语义**和**即时反馈**的颜色。它们不同于 **[[2.1.1 - 品牌色规范.md|品牌色]]**，主要目的是引导用户行为、提示状态变化和营造促销氛围。

**核心目标：**

1.  **传达清晰语义：** 利用色彩的通用心理暗示（如红色代表危险/紧迫，绿色代表成功/安全），实现高效的非语言沟通。
2.  **强化交互反馈：** 配合 [[5.1.2 - 状态变更反馈.md|微交互]]，为用户的操作提供即时的、视觉显著的响应。
3.  **驱动转化：** 利用高饱和度的营销色，实现 **[[1.1.1 - 视觉显著性]]**，刺激用户决策。

### 🚨 功能色规范：状态反馈的标准色

功能色用于在界面中清晰地表示系统的状态或用户操作的结果，必须具备极高的语义清晰度。

| 功能色用途 | 颜色象征与常用 HEX 值 | 视觉表现与语义 | 电商应用场景 |
| :--- | :--- | :--- | :--- |
| **成功（Success）** | **🟢 成功绿**（如 #4CAF50 或 #00B894） | 确认、操作完成、安全、支付成功。 | 支付成功提示、表单校验通过、商品已收藏。 |
| **警告（Warning）** | **🟡 警告黄**（如 #FFC107 或 #FFA000） | 轻微问题、需要注意但非致命的错误。 | 弱提示（如运费险即将过期）、低库存提醒。 |
| **错误/危险（Error/Danger）** | **🔴 警告红**（如 #F44336 或 #D32F2F） | 致命错误、操作失败、需要立即停止或修复。 | 密码输入错误、订单创建失败、账号异常提示。 |
| **提示/信息（Info）** | **🔵 提示蓝**（如 #2196F3 或 #1890FF） | 通用通知、信息提示、加载中状态。 | 系统更新通知、Toast 提示、帮助中心链接。 |

### 🔥 营销色规范：加速转化的情绪色

营销色通常是高饱和度、高显著性的颜色，用于营造紧迫感和优惠氛围。它们常用于 **[[1.2.1 - 紧迫感构建.md|紧迫感构建]]** 和 **[[1.2.3 - 价格锚定视觉化]]**。

| 营销色用途 | 颜色象征与常用 HEX 值 | 视觉表现与语义 | 电商应用场景 |
| :--- | :--- | :--- | :--- |
| **促销主红** | **🔥 促销红**（如 #FF5722 或 #E53935） | 紧迫、热烈、血本优惠、限时抢购。 | 秒杀倒计时、核心 CTA 按钮（“立即抢购”）、优惠券角标。 |
| **高亮橙/黄** | **💰 优惠橙**（如 #FF9800 或 #FF8C00） | 财富、利益、补贴、引人注目。 | 百亿补贴标签、价格锚定的现价高亮、金币/积分图标。 |

### 📐 应用与适配规范

1.  **WCAG 标准保障：** 所有的功能色，尤其是在作为文字背景色使用时，必须通过 **[[2.1.3 - 色彩对比度（WCAG）]]** 检测，确保即使是色盲用户也能准确识别状态。
2.  **避免滥用：** 营销色和警告色具有高显著性，应严格限制使用范围和比例。过度使用会导致**视觉疲劳**和**注意力分散**，降低其有效性。
3.  **深色模式适配：** 在 [[2.1.4 - 深色模式适配.md|深色模式]] 下，功能色的**饱和度需要降低**，但**明度对比度必须保持**，以确保其语义（Success/Error）依然清晰。
4.  **微交互协同：** 功能色应与 [[3.3.3 - 模态与非模态视图.md|弹窗]]、Toast 提示等容器设计协同，例如：支付成功的 Toast 提示应使用**成功绿**的背景或图标。

### 🛒 电商案例：拼多多与微信支付

* **拼多多促销按钮：** 拼多多的核心转化按钮（如“拼单”）大量使用高饱和度的**促销红**或橙色，结合 **[[1.1.3 - 拇指法则与热区设计.md|热区]]** 布局，强化用户的即时行动意愿。
* **微信支付成功提示：** 支付流程的最终反馈，使用 **成功绿** 的图标和文字（如“支付成功”）来传达安全、顺利完成交易的状态，建立用户信任。
