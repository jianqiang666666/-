## 📄 2.3.3 - 流媒体视觉规范.md

标签：#图像系统 #视频设计 #电商直播 #基础元素

### 🎥 四级子项：定义与核心逻辑：短视频与直播内容的视觉化标准

**流媒体视觉规范（Streaming Media Visual Specification）** 专指针对电商平台中日益重要的短视频、直播、动态 Banner 等动态内容所制定的视觉标准。这些动态元素具有高信息密度和强沉浸感，是 **[[1.1.1 - 视觉显著性]]** 的最高表现形式。

**核心目标：**

1.  **提升吸引力：** 利用动态影像的优势，快速捕获用户注意力并传达商品信息。
2.  **统一用户体验：** 确保视频播放器 UI、直播间信息流和动态内容的展示效果与平台整体 **[[2.1.1 - 品牌色规范.md|品牌调性]]** 统一。
3.  **适配移动场景：** 针对移动端竖屏、全屏和流量限制等特性进行优化。

### 🎞️ 视频内容与格式规范

| 规范项目 | 标准参数 | 视觉与内容要求 | 电商应用场景举例 |
| :--- | :--- | :--- | :--- |
| **视频尺寸比例** | **9:16 (竖屏)** 或 **3:4** | 核心电商短视频（如商品主图视频）应优先采用竖屏比例，适配移动端全屏播放。 | 抖音/快手电商短视频、商品主图视频。 |
| **视频时长** | **15 - 60 秒** | 需在**前 3 秒内**突出商品的**核心卖点**或使用场景，遵循 [[1.1.2 - 视线流设计.md|用户扫视习惯]]。 | 快速种草、商品功能演示视频。 |
| **分辨率** | **≥ 1080P** (1920x1080) | 保证视频画质清晰，尤其是在展示商品细节和材质时，影响用户的 **[[1.2.2 - 信任背书设计.md|感知信任]]**。 | 商品详情页的高清视频播放。 |
| **画面内容** | **去文字化**（高优先级） | 避免在视频画面主体上添加过多文字或营销贴图，文字应通过**字幕**或 **UI 浮层**展示。 | 强调画面内容的专业性和 **[[1.3.1 - 本能层设计.md|美感]]**。 |

### 📺 直播间 UI 浮层设计规范

直播间需要大量信息同时展示（商品、评论、互动），UI 浮层的设计至关重要：

1.  **信息层级：** 必须遵循 **[[2.2.3 - 信息层级排版法.md|信息层级排版法]]**，将**主播画面**作为背景主体，**商品卡片**和**核心购买按钮**作为一级信息，**评论/互动**作为二级信息。
2.  **透明度与色彩：** UI 浮层（如点赞数、评论区背景）应使用**半透明**的 **[[2.1.1 - 品牌色规范.md|中性色]]**（如浅黑或浅灰），以最小化对视频内容的干扰。
3.  **热区避让：** **核心购买按钮**和**商品链接**必须放置在 [[1.1.3 - 拇指法则与热区设计.md|拇指舒适区]]，并避让系统手势区。
4.  **动效与反馈：** 互动反馈（如用户送礼、点赞）应使用 **[[1.3.2 - 惊喜感视觉元素.md|趣味化动效]]**，但需限制其持续时间，避免遮挡商品展示。

### 🛒 电商案例：商品主图视频与直播间商品橱窗

* **商品主图视频：** 相比静态图，主图视频能利用 **[[1.2.1 - 紧迫感构建.md|声音和运动]]** 元素，在用户滑过列表时瞬间吸引目光，有效提升点击率。
* **直播间商品橱窗：** 商品卡片在直播画面中通常是**固定悬浮**或**侧滑展示**，它的设计必须使用 [[2.1.2 - 营销色与功能色.md|高对比度的营销色]] 突出价格和库存，引导用户**在直播流内完成转化闭环**。
* **画质自适应：** 平台需根据用户网络状况自动调整视频码率，并在低速时提供清晰的 [[5.1.2 - 状态变更反馈.md|加载状态]] 提示，确保用户体验流畅。
