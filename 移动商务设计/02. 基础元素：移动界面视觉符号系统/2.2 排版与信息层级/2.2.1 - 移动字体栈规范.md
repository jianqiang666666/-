## 📄 2.2.1 - 移动字体栈规范.md

标签：#排版系统 #字体规范 #可读性 #基础元素

### 🔡 四级子项：定义与核心逻辑：跨平台一致性与可读性的保障

**移动字体栈规范（Mobile Font Stack Specification）** 是指为 App 定义一套跨平台、多语种的字体使用优先级列表。它确定了在 iOS、Android 和 Web 等不同环境下，**系统字体和定制字体**的加载顺序和显示规则。

**核心目标：**

1.  **保障一致性和可读性：** 确保用户在任何设备上都能以最高的清晰度阅读文本，同时保持品牌视觉的统一性。
2.  **性能优化：** 优先使用系统原生字体，以避免加载字体文件带来的性能损耗。
3.  **多语种支持：** 确保字体栈能够平稳过渡，支持中文、英文、数字及其他必要的语种。

### 💻 字体栈构成：系统优先原则

为保证最佳的兼容性和性能，移动电商应用应优先使用各平台的原生字体：

| 平台 | 字体名称 | 字体栈优先级 | 理由与特性 |
| :--- | :--- | :--- | :--- |
| **iOS / macOS** | **PingFang SC** (苹方) / **SF Pro** | **第一优先级** | 原生渲染优化，内置多种字重，清晰度高，无需下载字体包。 |
| **Android** | **HarmonyOS Sans** (华为) / **MiSans** (小米) / **Roboto** (Google 原生) | **第二优先级** | 各手机厂商深度定制的字体更贴合系统体验，其次是 Roboto。 |
| **定制/品牌字体** | **（如：淘宝体）** | **第三优先级**（谨慎使用） | 仅用于 Logo、核心标题、价格等 **[[1.1.1 - 视觉显著性.md|高显著性]]** 元素，且需按需加载以避免性能问题。 |
| **数字/价格专用** | **DIN / FZLTTH** | **高优先级**（非系统） | 具有更好的数字辨识度、平衡感和视觉冲击力，常用于 **[[1.2.3 - 价格锚定视觉化.md|价格]]** 突出。 |

### 🛠️ 实操规范：字重与字体的精确应用

1.  **字重（Font Weight）规范：**
    * **Regular (400)：** 用于**正文**、商品描述等大段落文本。
    * **Medium (500) / Semibold (600)：** 用于**次级标题**、**列表项重点**、功能按钮，以实现 **[[2.2.3 - 信息层级排版法.md|信息层级]]**。
    * **Bold (700)：** 仅用于**一级标题**、**核心价格**、**警示信息**等最高层级的 [[1.1.1 - 视觉显著性.md|显著性]] 元素。
2.  **字体粗细与行高：** 必须与 [[2.2.2 - 字号与行高规范.md|字号和行高]] 协同设计。字重越粗，行高应适当增加，以保证可读性。
3.  **价格字体使用：** **[[1.2.3 - 价格锚定视觉化.md|价格]]** 数字应使用**专用的非衬线等宽字体**（如 DIN 或 FZLTTH）以确保在视觉上与正文区分，并强化其冲击力。

### 🛒 电商案例：数字字体的特殊处理

* **天猫/淘宝价格数字：** 采用定制的数字字体（如淘宝体或其变体）来渲染价格。这种字体通常笔画较粗、数字结构稳固，能够在复杂的商品卡片和营销活动中，实现**极高的视觉显著性**。
* **数字符号对齐：** 数字字体栈中，应确保货币符号（¥）和单位（万、件）与数字本身保持**基线对齐**，尤其在粗体显示时，要避免视觉上的不协调感。
* **促销倒计时：** 在 **[[1.2.1 - 紧迫感构建.md|倒计时模块]]** 中，数字应使用全角数字或**等宽数字**，确保数字变化时，模块宽度不会跳动，维持视觉稳定性。
